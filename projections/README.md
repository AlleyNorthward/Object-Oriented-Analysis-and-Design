# 简介

这部分会存贮我模拟的一些项目, 用于参考. 实际上, 大家其实也都可以抽象模拟. 但是, 你需要明确, 你是否真的认为这是你抽象的一个项目, 还是仍旧以做题的思维来思考, 这是非常不同的.  

比如第一个, `cpu-registers`, 这是小项目, 是我在模拟编写操作系统时, 想到的. 当时是期望有实际东西, 能够动态地展示`cpu`运行时, 内部寄存器是如何变化的. 但是分析后, 发现, 用`python`来抽象模拟, 还是很好的. 在此呢, 发现自己面向对象设计的问题, 所以才有了目前这个的面向对象设计与分析总结的项目. 

# 架构

- project
    - all-versions
        - 1.0
        - 2.0
        - 3.0
    - best-version
    - details
        - connections
        - objects

整体架构, 感觉就这样吧. 其中, `all-versions`, 存储着该项目的所有代码, 便于后续比较分析. 当然, 这里面不一定是全部的代码, 有的可能只是针对某一个对象的代码. 关于用到了什么设计模式, 会在`README.md`中详细描述的. 然后`best-version`, 是当下最优的版本. 如果有更好的设计方案, 会替代这个最优方案, 并放到`all-versions`中, 并且给出详细解释, 为什么会被替换. `details`则是整个项目的细节. 由于是细节部分, 所以后续会再进行补充的, 而不只是现在固定的样子. `objects`呢, 描述的是所有对象. `connections`呢, 则是所有对象之间的联系.
